2. INSTRUCTIONS D'EXÉCUTION
========================================

Voir le fichier README.md dans le repository pour les instructions détaillées.

MÉTHODE RAPIDE (Docker Compose):
1. Cloner le repository: git clone https://github.com/Rh-Man/TodoApi.git
2. Naviguer dans le dossier: cd TodoApi
3. Lancer: docker-compose up --build
4. Accéder à: http://localhost:5000/swagger

MÉTHODE LOCALE:
1. Démarrer MongoDB: docker run -d -p 27017:27017 --name mongodb mongo:latest
2. Naviguer dans TodoApi: cd TodoApi
3. Lancer l'API: dotnet run
4. Accéder à: http://localhost:5110/swagger

3. FONCTIONNALITÉS RÉALISÉES
========================================

✅ API Web ASP.NET Core 10.0 avec MongoDB
   - CRUD complet pour les Todos
   - Architecture en couches (Controllers, Services, Models)
   - Base de données MongoDB avec MongoDB.Driver

✅ Authentification JWT
   - Inscription d'utilisateurs avec hachage BCrypt
   - Connexion avec génération de tokens JWT
   - Tokens avec expiration (7 jours)
   - Claims: ID utilisateur, nom, email, rôle

✅ Autorisation par Rôles
   - Rôle Admin: Peut créer, modifier, supprimer des Todos
   - Rôle User: Peut uniquement lire les Todos
   - Attributs [Authorize] et [Authorize(Roles = "Admin")]

✅ Sécurité
   - Mots de passe hachés avec BCrypt (salt automatique)
   - Tokens JWT signés avec HMAC-SHA256
   - Validation des tokens sur chaque requête
   - Protection contre la sur-publication (overposting)
   - HTTPS en production

✅ Déploiement Docker
   - Dockerfile multi-stage optimisé
   - Docker Compose avec MongoDB
   - Variables d'environnement configurables
   - Volumes persistants pour les données

✅ Documentation
   - README.md complet avec exemples
   - Swagger UI intégré (/swagger)
   - Collection Postman (TodoApi.postman_collection.json)
   - Guide de déploiement pour plusieurs plateformes
   - Scripts de test PowerShell

7. ENDPOINTS DE L'API
========================================

Authentification (Public):
- POST /api/auth/register - Inscription
- POST /api/auth/login    - Connexion

Todos (Authentifié):
- GET    /api/todos       - Lire tous (User + Admin)
- GET    /api/todos/{id}  - Lire un (User + Admin)
- POST   /api/todos       - Créer (Admin uniquement)
- PUT    /api/todos/{id}  - Modifier (Admin uniquement)
- DELETE /api/todos/{id}  - Supprimer (Admin uniquement)
